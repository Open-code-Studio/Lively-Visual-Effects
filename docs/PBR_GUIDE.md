# PBR 材质格式详细指南

> 本文档详细说明 Minecraft Iris Shaders 使用的 **labPBR 1.3** 貼图格式规范。

---

## 一、文件命名规则

```
<基础贴图名>.png      # 反照率/漫反射贴图
<基础贴图名>_n.png    # 法线贴图
<基础贴图名>_s.png    # 镜面/金属/发光贴图
```

示例：
```
stone.png       →  基础颜色
stone_n.png     →  法线贴图
stone_s.png     →  规格贴图
```

> ⚠️ **文件名大小写敏感**，必须与 Minecraft 原版贴图文件名完全一致。

---

## 二、法线贴图 `_n.png`

### 格式要求
- **格式**: PNG, 24位/32位 RGB
- **空间**: 切线空间 (Tangent Space)
- **朝向**: **OpenGL 格式**（Y 轴向上）
- **默认值**: RGB(128, 128, 255) 表示完全平面

### 生成方法
1. 从高度图使用 Sobel 算子计算法线
2. 使用专业工具（Substance, Materialize, AwesomeBump）
3. 使用项目提供的 `generate_pbr.py` 脚本自动生成

### 法线方向对照表

| 朝向 | RGB 值 | 说明 |
|------|--------|------|
| 平面 | (128, 128, 255) | 完全平坦 |
| 向上 | (128, 255, 128) | 凸起区域（被光照亮） |
| 向右 | (255, 128, 128) | 右侧倾斜 |
| 向下 | (128, 0, 128) | 凹陷区域 |
| 向左 | (0, 128, 128) | 左侧倾斜 |

## 三、镜面/金属/发光贴图 `_s.png`

### 通道分配

| 通道 | 属性 | 范围 | 默认值 |
|------|------|------|--------|
| **R** | Smoothness（光滑度） | 0-255 | 0 |
| **G** | Metalness（金属度） | 0-255 | 0 |
| **B** | Emission（自发光） | 0-255 | 0 |
| **A** | Subsurface Scattering（次表面散射） | 0-255 | 0 |

### 各属性详解

#### R 通道 — 光滑度 (Smoothness)
控制表面微观粗糙度，影响反射的清晰程度。

- `0`: 极度粗糙（如泥土）— 漫反射为主，反射模糊
- `128`: 中等光滑（如打磨过的石头）
- `255`: 完全光滑（如镜子）— 清晰的镜面反射

> **注意**: 部分光影使用 `Roughness`（粗糙度）而非 `Smoothness`。
> `Roughness = 255 - Smoothness`。本项目采用 **Smoothness** 标准。

#### G 通道 — 金属度 (Metalness)
控制材质的金属属性。

- `0`: 完全非金属（石头、木头、玻璃）
- `255`: 完全金属（铁、金、铜）
- 中间值: 部分金属化（如含矿岩石）

> 金属材质的 F0（基础反射率）由 albedo 颜色决定；
> 非金属材质的 F0 约为 0.04（约 4% 反射率）。

#### B 通道 — 自发光 (Emission)
控制材质自发光强度，不受环境光照影响。

- `0`: 不发光（正常材质）
- `128`: 中等亮度（如红石灯关闭状态）
- `255`: 全亮（如萤石、岩浆）

## 四、贴图分辨率

| 质量等级 | 分辨率 | 适用场景 |
|----------|--------|----------|
| 标准 | 16×16 | 低配电脑 |
| HD | 32×32 | 中等配置 |
| HD+ | 64×64 | 高配电脑 |
| UHD | 128×128+ | 极高配置 |

> 推荐与基础贴图保持相同分辨率。

## 五、常见问题

### Q: PBR 效果不显示？
1. 确认光影包支持 labPBR（如 Complementary、BSL、SEUS）
2. 在视频设置中启用法线贴图和镜面贴图选项
3. 检查文件名是否完全匹配

### Q: 材质看起来太亮/太暗？
- 检查法线贴图是否为 OpenGL 格式（Y+ 向上）
- 调整 `_s.png` 的 R 通道光滑度值

### Q: DirectX 和 OpenGL 法线有什么区别？
- **OpenGL**: Y 通道向上（绿色通道表示上方）— Iris 默认使用
- **DirectX**: Y 通道向下（绿色通道表示下方）
- 可通过翻转法线贴图的 G 通道来转换
